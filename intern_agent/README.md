# Агент фильтрации junior-вакансий и стажировок.

Проект (Вариант B): rule-based фильтрация + LLM-ранжирование стажировок и junior-позиций по mock-данным без парсинга job-сайтов.

## Проблема

Студенту сложно вручную просматривать десятки вакансий и отделять релевантные junior/стажёрские позиции с нужным стеком и форматом работы.

## Pipeline

```
vacancies.csv → pandas-фильтры → criteria.md → Groq LLM → report.md
```

**Rule-based шаги (обычная логика):**

- удаление битых и пустых строк;
- dedup по `id`;
- `is_junior == True`;
- удалёнка (`Remote` / `Удаленно`);
- навыки: Python / C++ / алгоритмы;
- свежесть: не старше 90 дней.

**LLM-шаг (агентность):**

- получает уже отфильтрованный список и критерии из `criteria.md`;
- ранжирует **топ-5**;
- для каждой позиции пишет: почему подходит, требования, что смущает.

Если LLM недоступен (ключ, лимит, сеть) — создаётся **fallback-отчёт** без падения скрипта.

## Запуск

```bash
pip install -r requirements.txt
```

Создайте `.env`:

```env
GROQ_API_KEY=gsk_ваш_ключ
GROQ_MODEL=llama-3.3-70b-versatile
```

```bash
python main.py
```

Результат: `report.md`. Логи — в консоли (уровень INFO).

## Файлы


| Файл            | Назначение                                            |
| --------------- | ----------------------------------------------------- |
| `vacancies.csv` | Mock-вакансии (15 строк, включая битые/нерелевантные) |
| `criteria.md`   | Критерии кандидата                                    |
| `main.py`       | Агент: фильтрация + LLM + отчёт                       |
| `report.md`     | Итоговый Markdown-отчёт                               |
| `.env`          | API-ключ Groq                                         |


## Ограничения и риски

- Данные mock, не live-парсинг hh/LinkedIn.
- Rule-based фильтр навыков — по ключевым словам, возможны ложные срабатывания.
- LLM может «додумывать» детали — нужна ручная проверка перед откликом.
- Groq имеет rate limits; при ошибке используется упрощённый fallback.

## Что было сложнее всего

Разделить ответственность: жёсткие правила для отсева мусора и LLM — только для осмысленного ранжирования и объяснений, плюс graceful degradation при сбоях API.

```csv

12,Senior Software Architect,BigTech,Москва,350000,"10+ лет опыта",False,2026-06-01

Фрагмент `criteria.md`:

Я студент, ищу стажировку или junior-вакансию.
Знаю математику, C++, базовый Python, алгоритмы. Интересует удаленка.
```
